On prend les commandes

Conditions d’achèvement

Unix

  • UNIX: système d’exploitation multi-tâches et multi-utilisateurs créé en 1969, conceptuellement ouvert et fondé sur une approche par laquelle il offre de nombreux petits outils chacun dotés d’une mission spécifique.

  • Il a donné naissance à une famille d’environ 25 systèmes, dont les plus populaires en 2015 sont GNU/Linux, BSD, IOS et Mac OS X. On parle de « famille Unix » pour l’ensemble de ces systèmes.

  • Base d’Internet: protocole TCP/IP 1982

GNU/Linux

  • Linux 1991 système d’exploitation compatible POSIX, logiciel libre, noyau qui utilise les logiciels GNU de la Free Software Fundation

    • Nombreuses distributions Linux indépendantes
      Les distributions GNU/Linux au cours du temps

      -> trouver d'où vient votre distribution et d'où vient Android. Quelles sont les plus anciennes version de GNU/Linux ? Quelles sont les grandes familles ?

L’arborescence des fichiers

Système de fichiers

Définition

  • Un système de fichiers sert à stocker et organiser l’information sur des mémoires de masses.

  • Il permet de conserver un grand nombre de données, les partager avec différent programme.

  • Les principales actions sont

    • Créer/Détruire

    • Allouer de la place en mémoire

    • Localiser les fichiers

    • Assurer la sécurité et le contrôle des fichiers

L’arborescence des fichiers

L’utilisateur voit le système de fichier comme une arborescence :
les fichiers sont regroupés dans des répertoires qui contiennent eux-mêmes soir des fichiers soit des répertoire.
Chaque fichier et répertoire est accessible par un chemin

Sous Unix : « Tout est fichier »

Les répertoires, les programmes, les périphériques sont considérés comme des fichier.

  • racine unique /

  • tout support (disque, clé USB, etc.) est greffé (ou monté) sur l’arborescence courante,

  • il existe un chemin unique de la racine vers n’importe quel fichier sur n’importe quel disque,

  • tout répertoire ou fichier a un nom

  • un chemin est une suite de noms séparés par des /.

L’arborescence à Linux

Arborescence de fichiers GNU/Linux

  • etc : contient des fichiers des paramètres systèmes.

  • dev : fichiers dits périphériques.

  • bin : contient les programmes de base du système.

  • home : contient les répertoires et fichiers alloués à chaque utilisateur.

  • lib : librairies.

  • local : généralement situé dans le répertoire usr contient l’ensemble des programmes spécifiques à une machine.

  • usr : contient la majeure partie du système.

  • var : contient des informations sur le fonctionnement du système.

Adresses

Adresse Absolue  : à partir de la racine de l’arborescence

Exemples : /home/sasl/shared/main/essai
/home/sasl/encad/braunstein/controle3.pdf

Adresse Relative : à partir de où je me trouve dans l’arborescence

Exemples : Images/Ete2015/IMG150728.jpg
../Mes Documents/mon_rapport.pdf

My Home directory

Son home directory  ou son chez soi est l'adresse à laquelle on "ouvre" son dossier personnel, c'est aussi l'adresse sur laquelle on arrive lorsque l'on ouvre un terminal. Il est général de la forme /home/login où login est votre nom de compte sur la machine.

Les métadonnées d’un fichier

Un fichier a toujours un nom, un contenu mais il possède aussi d’autres informations

  • Droits d’accès (r/w/x)

  • Dates de derniers accès

  • Propriétaire

  • Taille

  • Type de fichier

L’interpréteur de commandes

Le terminal


Interprète de commande : Un programme qui attend que l’on tape des commandes et affiche le résultat. L'attente se voit avec un curseur clignotant.

L’interprète de commande : un langage de programmation spécialisé

Un langage de programmation est fait pour programmer un certain type d’applications

Un interprète de commandes tel que bash est:

  • spécialisé dans le contrôle de processus, la manipulation de flux et de fichiers,

  • optimisé pour l’écriture de commande d’une seule ligne: lancement efficace d’un programme dont on connaît le nom truc

Un interprète de commandes apporte une boucle d’interaction (toplevel) d’où le prompt qui indique l’attente de la commande suivante

Le cas particulier de l’interprète de commandes (mise en œuvre standard)

Le terminal est un processus comme les autres avec un environnement d’exécution (voir les variables d'environnement pour plus information).

stdin : flux d''entrée standard (le clavier)

stdout : flux de sortie standard (l'écran sur le terminal)

stderr : flux de sortie d'erreur standard (l'écran sur le terminal)

Exécution de commande

Exécution de la commande date : un processus est crée le calcul est fait et la main est rendu au terminal, ça va trop vite pour le voir lorsque vous essayer de taper la commande.


L’appel de commande

Interprète de commandes: syntaxe de l’appel de commande

  • La ligne de commande est découpée en « mots ».

  • Le premier mot désigne la commande à exécuter.

  • Les mots suivants peuvent être des options (avec un ou deux tirets en préfixe), des noms de fichiers ou de simples chaînes de caractères.

  • commande argument1 ’argument 2’ arg’ trois

    ls --color=never -l /var
man -k cat
cat -n macros.sty
ps --help

Syntaxe d’une commande

  nom_commande [options] [arg1 .. argN]

  • Les options sont précédées du signe -

  • Les paramètres optionnels sont représentés entres crochets

Où se trouve la commande?

  • Si le nom de la commande contient / alors c’est un nom de fichier (absolu ou relatif).

    /bin/ls
./a.out

  • Si la commande ne contient pas de / alors c’est :

    • soit un programme à chercher dans les répertoires figurant dans la variable PATH.

      % printenv PATH
/usr/local/X11/bin:/usr/bin/X11:/usr/local/bin:/usr
/bin:/bin:/usr/sbin:/etc:/usr/etc

    • soit uneprimitive (cd, umask, export, source, etc...) qui ne peut pas être écrite comme un programme parce qu’elle apporte une modification qui doit affecter le shell qui l’exécute et non son clone momentané créé pour l’exécution d’une commande

La commande man

Besoin d’aide

  • Obtenir de l’aide sur une commande

    $ man [num_section] nom_commande

  • Obtenir une information succinte sur une commande

    $ whatis nom_commande
$ nom_commande --help

  • Trouver une commande selon un mot clé

    $ man -k mot_clé

Variable d’environnement

Variables d’environnement

Les variables d’environnement communiquent des informations au programme qui s’exécutent.

  • HOME, USER

  • SHELL, EDITOR

  • DISPLAY

  • PATH

  • BROWSER

  • LANG

  • HOSTNAME

  • …

Créer/modifier des variables

  • syntaxe (création/modification): VARIABLE=valeur,
    syntaxe (rendre exportable): export VARIABLE,
    syntaxe (création/modification et exportation): export VARIABLE=valeur

  • la variable VARIABLE, à gauche du signe =, lvalue

  • la valeur associée $VARIABLE, à droite du signe =, rvalue

  • visualisation printenv VARIABLE ou echo $VARIABLE

%echo LANG
LANG
%echo $LANG
fr_FR.utf8
%LANG=en_EN.utf8
%export LANG=en_EN.utf8

Commentaires

  • En [ba]sh, syntaxe précise: pas de blanc autour du signe =,

  • Blanc après le signe =: lié à rien,

  • Blanc avant le signe =: lancer le programme VARIABLE avec un argument =valeur

  • printenv: imprimer pour l’environnement [shell]POSIX dans lequel on se trouve, la liste de toutes les valeurs qui sont définies à ce moment-là.

  • Il est possible de changer les valeurs automatiquement à l’ouverture du shell en ajoutant des exports dans le .bashrc.

Exploration de l’arborescence

Exploration de l’arborescence

  • pwd (Print Work Directory), cd (Change Directory), mkdir (MaKe DIRectory), rmdir (ReMove DIRectory)

  • ls (LiSt, affiche le contenu d’un répertoire), rm (ReMove), supprime les fichiers passés en argument

  • cp (CoPy), mv (MoVe, renommage et déplacement)

  • file (type d’un fichier), cat (conCATenate, concaténer des fichiers et les afficher sur la sortie standard), more et less (filtres lecteurs de fichier)

  • commandes largement vues en TP avec leurs principales options

  • quelques exemples courants avec options courantes:

    {style=custombash}
mkdir -p
rm -r -f -i
ls -l -a -t -r

Parcours de l’arborescence de fichiers sous Unix

Exemples avec les commandes cd Change Directory et pwd Print Working Directory

% cd /home/cecile/Enseignement/
% pwd
/home/cecile/Enseignement/
% cd .

% pwd

Droits des fichiers

Droits

  • Tout fichier (y compris les répertoires) appartient à un utilisateur et à un groupe

  • Un utilisateur appartient à au moins un groupe

  • Visualisables par ls -l

    drwxrwxr-x 3 cecile users  4096 jan 31 21:58 CQ
-rw-rw-r-- 1 cecile users  4912 nov 12 11:15 envdev\_reunion1
lrwxrwxrwx 1 cecile users    19 fév 17 14:08 envdev.sty -> ../Style
                                                          /envdev.sty

Détails sur la sortie de ls -l

  • Les dix premiers caractères pour les droits, puis le nombre de liens physiques, le nom du propriétaire et du groupe, la taille en octets, l’horodatage (par défaut date de dernière modification).

  • Les dix caractères:

    • le premier caractère: le type du fichier

    • les neuf suivants: les permissions d’accès: 3 caractères pour le propriétaire puis 3 pour le groupe et 3 pour les autres.

  • Le type du fichier:

    -
    pour les fichiers ordinaires,
    d
    pour un répertoire (directory),
    l
    pour un lien symbolique,
    b,c,p,s
    fichiers spéciaux, pas détaillé ici

Permissions d’accès

  • Représentation symbolique

    r
    pour avoir le droit de lire (Read)
    w
    pour avoir le droit d’écrire (Write) ou de supprimer
    x
    pour avoir le droit d’exécuter un fichier (eXecute) ou de parcourir un répertoire
    s
    pour exécuter le fichier comme le propriétaire ou le groupe propriétaire du fichier (intéressant si par exemple il s’agit de root)

  • - signifie: pas le droit correspondant à la position

  • Droits modifiables par chmod

    chmod go+r fich1 
chmod -R u+x,o-w dir1 fich2
chmod a-rw fich3

Les permissions d’accès symboliques

  • Syntaxe de chmod pour les permissions d’accès symboliques: `[ugoa]*([-+=]([rwxXst]*|[ugo]))+'

  • + pour ajouter le droit, - pour le retirer, = pour n’autoriser que l’accès indiqué

  • paramètre global -R récursif

  • personnes concernées:

    u
    User
    g
    Group
    o
    Others
    a
    All
